## What is agent-journal?

Agent-journal captures the behavioral trace of a Claude Code SDK session — tool calls, LLM calls, token costs, phase boundaries — as a structured, append-only event log for offline analysis.

It is to agent research what a flight data recorder is to aviation safety: the raw record of what happened, in what order, at the right granularity.

```xml
<dependency>
    <groupId>io.github.markpollack</groupId>
    <artifactId>journal-core</artifactId>
    <version>0.1.0-SNAPSHOT</version>
</dependency>
```

## The core abstraction: Experiment / Run

Agent execution is not a trace (spans with parent-child timing). It is a **run** — a bounded execution with:

| Part | Description |
|------|-------------|
| **Config** | Immutable inputs set at run start (model, task, agent name) |
| **Events** | Append-only log of what happened (tool calls, LLM calls, state changes) |
| **Metrics** | Named measurements accumulated during the run |
| **Summary** | Mutable outputs recorded at the end |

Runs belong to **Experiments**. An experiment answers a question ("does this prompt reduce JAR_INSPECT calls?"). Runs are the repeated trials.

## What gets captured

Every run produces a typed event log:

- `LLMCallEvent` — model, token counts (input/output/thinking/cache), cost breakdown, latency
- `ToolCallEvent` — tool name, input parameters, duration, success/failure
- `StateChangeEvent` — named state transitions as the agent progresses
- `GitCommitEvent`, `GitPatchEvent`, `GitBranchEvent`, `GitPullRequestEvent` — code changes
- `MetricEvent`, `CustomEvent` — measurements and domain-specific events

This log persists as JSONL — one event per line, append-only — enabling downstream analysis with DuckDB, Parquet pipelines, or Markov chain analysis of tool-call sequences.

## What it is NOT

- **Not an observability tool.** OpenTelemetry is designed for distributed production monitoring — spans, collectors, dashboards, alerting. Agent-journal is for deep per-session research analytics.
- **Not a platform.** No server, no daemon, no database. A JAR you add to your project.
- **Not real-time.** Events are written for offline analysis, not live dashboards.
